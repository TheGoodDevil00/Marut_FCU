# Support

Use the repository documentation first, then choose the support channel that fits the problem.

## Before Opening an Issue

1. Read `docs/ARCHITECTURE.md`.
2. Read `CONTRIBUTING.md`.
3. Review the target `.ioc` and `main.c` you are working in.
4. Check whether the issue is already reported.

## Use Issues For

- confirmed bugs
- documentation gaps
- feature requests tied to the repository

## Use Discussion or Maintainer Contact For

- architecture questions
- target-selection questions
- proposal sanity checks before large work
- safety-related issues that should not begin in public

## Good Bug Reports Include

- affected target
- MCU / board
- toolchain version
- reproduction steps
- logs or serial output if available
- expected behavior
- observed behavior

## Scope

This is a student-built research and engineering project. Support is best-effort, and the clearest reports are the easiest to act on.
